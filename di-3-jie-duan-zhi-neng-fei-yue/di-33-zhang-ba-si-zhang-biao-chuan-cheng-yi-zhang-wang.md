# 第3-3章：把四张表"串"成一张网

“打开Power Pivot，在关系图视图中把【广告活动ID】从「广告日报」拖到「产品映射表」——一条连线，两张表关联上了。这就是数据建模。”

### 现在就动手

打开 「广告分析学习工作簿」 。点击顶部菜单，看看是否有 “Power Pivot” 选项卡。

如果没有，按以下步骤启用：

1\.        文件 → 选项 → 加载项

2\.        底部“管理”下拉选择“COM加载项”→点击“转到”

3\.        勾选 “Microsoft Power Pivot for Excel” → 确定

有了Power Pivot之后，在Excel中按 Alt → P → M，打开Power Pivot窗口。

切换到 “关系图视图” 。

在左侧的“表”列表中，找到 「广告日报」 。把鼠标放在【广告活动ID】这一列上，按住，拖到 「产品映射表」 的【广告活动ID】列上，松开鼠标。

你看到了什么？

一条连接两个表的线出现了。一端是 1，一端是 \*。

💡 在单元格里试试就知道了。

你刚刚建立了一个“关系”：告诉Power Pivot——两个表通过【广告活动ID】关联在一起。

现在，当你在透视表中使用数据模型时，你可以同时从两个表拖拽字段——产品线在「产品映射表」里，花费在「广告日报」里，但Excel知道它们是一家的。

这就是数据建模的核心：把分散的表，通过关系“串”成一张网。

### 一、Power Pivot是什么？

一句话概括：

Power Pivot是Excel内置的“数据库引擎”，让你在Excel中建立多表之间的关系。

为什么需要Power Pivot？

在第0阶段和第2阶段，你积累了大量的数据：

<table data-header-hidden><thead><tr><th width="119"></th><th width="260.66668701171875"></th><th width="112.77777099609375"></th></tr></thead><tbody><tr><td>表名</td><td>内容</td><td>行数</td></tr><tr><td>广告日报</td><td>每日广告投放明细</td><td>600行</td></tr><tr><td>产品映射表</td><td>产品线、负责人、目标ACOS</td><td>30行</td></tr><tr><td>日历表</td><td>年、月、季度、周</td><td>212行</td></tr><tr><td>渠道表</td><td>渠道代码、渠道全称、渠道类型</td><td>3行</td></tr></tbody></table>

这些表分散在不同的工作表里。以前你只有一种办法把它们“串”起来：用VLOOKUP/XLOOKUP把维度表匹配到事实表。

但当你需要按“月份”汇总时，你得把月份匹配到广告日报；按“产品线”汇总时，又得匹配产品线；按“渠道类型”汇总时，又得匹配渠道类型……每加一个维度，就得多加一列VLOOKUP。

Power Pivot的解决方案：把四张表导入一个“数据模型”，建立一次关系，之后你可以在透视表中任意组合这些表的字段——不需要写任何VLOOKUP。

事实表 vs 维度表

在数据建模中，表分为两种角色：

<table data-header-hidden><thead><tr><th width="81.22222900390625"></th><th width="207.4444580078125"></th><th width="318.22222900390625"></th></tr></thead><tbody><tr><td>角色</td><td>说明</td><td>广告示例</td></tr><tr><td>事实表</td><td>记录事件/交易（行数多）</td><td>「广告日报」（600行）</td></tr><tr><td>维度表</td><td>描述属性（行数少）</td><td>「产品映射表」（30行）、「日历表」（212行）、「渠道表」（3行）</td></tr></tbody></table>

关系：每个维度表都与事实表通过“关键字段”关联。

&#x20;                   ┌─────────────┐

&#x20;                   │  「日历表」   │

&#x20;                   │  (日期维度)  │

&#x20;                   └──────┬──────┘

&#x20;                          │

&#x20;   ┌─────────────┐   ┌────┴────┐   ┌─────────────┐

&#x20;   │「产品映射表」 │───│「广告日报」│───│ 「渠道表」   │

&#x20;   │ (产品维度)  │   │ (事实表) │   │ (渠道维度)  │

&#x20;   └─────────────┘   └─────────┘   └─────────────┘

这种结构叫 “星型模型” ——事实表在中心，维度表在周围。

### 五、DAX——计算语言

Power Pivot建立关系后，你需要在它之上“写公式”来计算指标。这个语言叫 DAX。

DAX vs Excel公式

度量值 vs 计算列

在DAX中，有两种方式创建计算：

<table data-header-hidden><thead><tr><th width="97.888916015625"></th><th width="189.4444580078125"></th><th width="195.111083984375"></th></tr></thead><tbody><tr><td>维度</td><td>计算列</td><td>度量值</td></tr><tr><td>计算时机</td><td>数据刷新时</td><td>查询时（透视表）</td></tr><tr><td>存储</td><td>占用内存</td><td>不存储，实时计算</td></tr><tr><td>适用场景</td><td>行级计算</td><td>聚合计算</td></tr><tr><td>示例</td><td>利润率 = 利润/销售额</td><td>总利润 = SUM(利润)</td></tr></tbody></table>

建议：优先使用度量值，节约内存，更灵活。

基础DAX度量值

在Power Pivot中，点击表格下方的“计算区域”，输入公式：

总花费 := SUM('广告日报'\[花费])

总点击 := SUM('广告日报'\[点击])

总订单 := SUM('广告日报'\[订单])

总销售额 := SUM('广告日报'\[销售额])

ROAS := DIVIDE(\[总销售额], \[总花费], 0)

ACOS := DIVIDE(\[总花费], \[总销售额], 0)

### 六、综合实战——用数据模型做跨表透视

操作步骤

Step 1：将四张表添加到数据模型

Step 2：在关系图视图中建立三个关系

Step 3：创建基础度量值（总花费、总销售额、ROAS、ACOS）

Step 4：返回Excel → 插入透视表 → 选择“使用此工作簿的数据模型”

Step 5：在透视表中，把【产品线】拖到行区域，把【总花费】拖到值区域

你看到了什么？

你使用了「产品映射表」的字段 + 「广告日报」的度量值，在同一个透视表中分析。不需要写任何VLOOKUP，因为关系已经在数据模型中定义好了。

你还可以继续拖拽：

· 把【月份】（来自日历表）拖到行区域 → 按月+产品线分析

· 把【渠道类型】（来自渠道表）拖到列区域 → 按渠道类型+产品线交叉分析

这就是数据模型的威力：一次建立关系，之后任意组合。
