\# 第1-1章：先刷新，再说话



“打开你搭建的模板，新增一行数据，点击刷新。你看到了什么？——3秒钟，所有数据更新完毕。这就是建模思维的威力。一次搭建，永久复用。”



\## 现在就动手



打开你在第0-8章搭建的 \*\*「广告日报模板」\*\* 。



在数据区域底部，你有汇总统计公式（比如总花费、总订单、整体ROAS）。



现在做一件事：在「广告日报」的最下方，随便新增一行数据（日期、渠道、广告活动ID、花费……随便填几个数字）。



然后，右键点击你的汇总统计单元格，选择 \*\*“刷新”\*\* （或按 Ctrl+Alt+F5）。



\*\*你看到了什么？\*\*



所有公式自动扩展、所有汇总自动更新、所有标签自动重算。



💡 \*\*在单元格里试试就知道了。\*\*



你只做了一个动作——新增一行数据 + 点击刷新。但Excel替你完成了：匹配产品线、计算ACOS、计算ROAS、打标签、拆分文本、提取日期、更新汇总……



\*\*这就是“建模思维”的力量：你搭建一次，Excel替你重复执行无数次。\*\*



如果你没有搭建模板，而是每次从头开始做，新增一行数据意味着：重新写公式、重新拖拽、重新汇总——5分钟的工作。



而你现在，3秒钟。



\*\*这一章，就是让你理解“建模思维”为什么是AI时代最重要的Excel能力。\*\*



\## 一、两种思维模式：你选哪一种？



\*\*传统思维：“每次重做”\*\*



| \*\*场景\*\* | \*\*做法\*\* |

| --- | --- |

| 拿到新数据 | 从零开始：导入→清洗→匹配→计算→汇总 |

| 老板问新问题 | 重新写公式、重新做表 |

| 数据源变了 | 手动改公式、改引用 |



\*\*特点\*\*：每次都在“造轮子”，重复劳动多，容易出错。



\*\*建模思维：“一次搭建，永久复用”\*\*



| \*\*场景\*\* | \*\*做法\*\* |

| --- | --- |

| 拿到新数据 | 粘贴数据→点击刷新 |

| 老板问新问题 | 在已有模型上拖拽字段 |

| 数据源变了 | 修改数据源路径→刷新 |



\*\*特点\*\*：第一次花时间搭建，之后每次用都只需几秒钟。



\## 二、AI时代为什么“建模思维”比“函数技巧”更重要？



在第0阶段，你学会了20个函数。这是你的 \*\*“工具箱”\*\* 。



但光有工具是不够的——你还需要知道：\*\*什么时候用哪个工具？怎么把它们组合成一条流水线？\*\*



\*\*函数技巧\*\* → 你会用锤子和钉子。  

\*\*建模思维\*\* → 你会盖一栋房子，而且下次只需要补几块砖，不用重新盖。



在AI时代，这个区别更加明显：



\- AI可以帮你写单个函数（锤子和钉子）

\- \*\*只有你\*\*能设计整个流程——数据从哪里来、经过哪些步骤、输出什么结果（房子的图纸）



\*\*你的核心竞争力，已经从“会写函数”变成了“会设计流程”。\*\*



\## 三、一个真实的对比



\*\*场景\*\*：每周一处理广告周报。



\*\*传统方式（没有建模思维）\*\*



1\.  从后台导出数据

2\.  用VLOOKUP匹配产品线（写公式，拖拽）

3\.  用IFS打标签（写公式，拖拽）

4\.  用SUMIFS做汇总（写公式）

5\.  创建透视表

6\.  调整格式

7\.  发给老板



\*\*每周耗时：30-45分钟。一年52周：26-39小时。\*\*



\*\*建模思维方式\*\*



\*\*第一次（3小时）\*\* ：



1\.  用Power Query建立清洗流程

2\.  用公式建立匹配、计算、标签

3\.  用透视表建立汇总

4\.  用切片器建立交互

5\.  把所有内容整合到一个工作簿



\*\*之后每周（3分钟）\*\* ：



1\.  粘贴新数据

2\.  点击刷新

3\.  复制图表发邮件



\*\*第一周投入3小时，之后每周节省40分钟。一年节省34小时。\*\*



\*\*这笔账，你算得过来。\*\*



\## 四、建模思维的三个核心原则



\*\*原则1：数据与计算分离\*\*



| \*\*坏习惯\*\* | \*\*好习惯\*\* |

| --- | --- |

| 在原始数据上直接写公式 | 原始数据只放数据，计算在公式列 |

| 每次修改数据都要改公式 | 公式引用的是列，数据变了公式自动算 |



\*\*原则2：一次定义，多次使用\*\*



| \*\*坏习惯\*\* | \*\*好习惯\*\* |

| --- | --- |

| 每次重新写VLOOKUP | 把公式写在一列，以后永远用这一列 |

| 每次重新建透视表 | 透视表建一次，之后刷新即可 |



\*\*原则3：自动化优先于手动\*\*



| \*\*坏习惯\*\* | \*\*好习惯\*\* |

| --- | --- |

| 手工筛选复制 | 用公式/透视表自动计算 |

| 每周重复同样操作 | 用Power Query记录步骤，一键刷新 |



💡 \*\*在单元格里试试就知道了。\*\* 你打开第0-8章的模板，新增一行数据，点击刷新——你亲手体验了“一次搭建，永久复用”。这不是理论，是你已经做到的事。



\## 五、AI能帮你什么？不能帮你什么？



\*\*AI能帮你\*\*



| \*\*能力\*\* | \*\*示例\*\* |

| --- | --- |

| 写单个公式 | “帮我写一个XLOOKUP” |

| 解释公式逻辑 | “这个公式是什么意思？” |

| 调试错误 | “为什么返回#N/A？” |

| 生成VBA代码 | “帮我写一段代码另存为PDF” |



\*\*AI不能帮你\*\*



| \*\*局限\*\* | \*\*说明\*\* |

| --- | --- |

| 设计整个流程 | 只有你懂业务逻辑 |

| 判断结果是否合理 | 只有你懂业务目标 |

| 选择用什么工具 | 只有你懂数据量和复杂度 |



\*\*AI是副驾驶，方向盘在你手里。\*\*



\## 六、这一章你带走了什么？



| \*\*你之前可能觉得\*\* | \*\*你现在知道了\*\* |

| --- | --- |

| 学会函数就行了 | 学会函数只是第一步，建模思维才是关键 |

| 每次从头做也挺快 | 一次搭建永久复用，长期省下几十小时 |

| AI能替我做所有事 | AI只能帮你写单个公式，设计流程还得靠你 |



\*\*一句话总结\*\*：



\*\*“建模思维”不是让你多学一个技能，而是让你换一种方式工作——从“每次重做”变成“一次搭建，永久复用”。\*\*



你已经做到了——你在第0-8章搭建的模板就是建模思维的产物。现在你要做的，是在后续章节中继续深化这个能力。
